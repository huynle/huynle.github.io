      Basics - Up and Running

      Understanding the order of operation for SQL was difficult to grasp initially. The following rough outline should help with that..

      First thing to note! SQL works from the inside clause out! The important relationship to note is the three following where row and columns data are being moved

      If SELECT:WHERE then GROUP BY:HAVING

      This simply means that WHERE is the filter used for SELECT and HAVING is the filter for GROUP BY. When filter is mentioned, imagine an expression that the filter have to meet; just like A > 10. If the expression is met, then the row gets to stays in the query.

      Another thing to note about SQL is the order of execution, it is fundamentally different from other programming languages. A regular functional programming language like that of Python of C++ would have the outer function run first. Then if it sees an inside function, that function is then executed and exit, and the outer function would be pick up what was evaluated. Python and C++, are performing actions, ie. functional programming. SQL, on the other end of the spectrum, it is storing and manipulating data, it is an ASKING language. Imagine a Venn diagram where each table is a circle, SQL allows us to ask for stack different tables together to form a result.

      SQLite have a lot of clauses and statements, the four clauses that were introduced are the basic clauses that you would need to get up and running.

      HAVING - This is a clause. It can only be use in conjunction with GROUP BY clause. HAVING follows by an expression that is evaluated per row from the table. GROUP BY - This is a clause to group the results of HAVING by whatever expression you choose. SELECT - This is a statement. This statement is an aggregates the results of the GROUP BY clause.

      Just from these four clauses. You can write you basic SQL to get almost anything you need. An example of this is.

      Clause Notes

      • GROUP BY
        • Group by will tell SQL to combine the unique groups into one row!
          • It will execute HAVING first before grouping
            --show how many of each Locus there is
  SELECT COUNT(*),Locus FROM Genes GROUP BY Locus

  --only show the groups that have more than one row in them
  SELECT COUNT(*),Locus FROM Genes GROUP BY Locus HAVING COUNT(*)>1

  --to just display all the rows for your condition, don't use GROUP BY or HAVING
  SELECT * FROM Genes WHERE Locus = '3' AND Chromosome = '10'

      Notes

      Relationship between GROUP BY and SELECT, the aggregate function in the SELECT statement uses the GROUP BY columns

      All columns in the SELECT clause that do not have an aggregate need to be in the GROUP BY

      -- good
SELECT col1, col2, col3, MAX(col4)
GROUP BY col1, col2, col3

-- Also good:
SELECT col1, col2, col3, MAX(col4)
GROUP BY col1, col2, col3, col5, col6

-- No other columns = no GROUP BY needed
SELECT MAX(col4)

-- Won't work:
SELECT col1, col2, col3, MAX(col4)
GROUP BY col1, col2

-- Pointless:
SELECT col1, col2, col3, MAX(col4)
GROUP BY col1, col2, col3, MAX(col4)
