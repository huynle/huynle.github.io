Vim Essentials
Apr 19, 2017
18 minute read

    There are so many things to talk about Vim, this article is aimed to help you get you familiar with Vim by learning what the basic features and ability of Vim.

    After using Vim for about a year now, there are a few things I wish I knew about Vim when I first made the commitment to jump over to this ancient looking editor; let me tell you, this text editor is anything be ancient. Active development are being done with Vim 8 and NeoVim. In addition to a plethora of plugins, this editor will fit all your text editing needs.

    Vim and NeoVim are basically the same, you can almost use it interchangeably. For this post, we will just call it Vim!

    Before diving into all the knobs you can turn with Vim, let’s concentrate on the essentials points to know about Vim. Unlike Notepad++ and Atom, Vim has a modal interface.

    Modes

    VIM have different modes

    • command
    • visual
    • insert

    Using VIM

    • Check mapping

      • :nmap command

    • Relative line movement

      • using 5j or 5k will move the cursor down/up, respectively

    • searching

      • :/foo will find foo
        • use n

    • marking line to come back to

      • ma to mark the line
        • a` to jump to line, `a` can be replaced by any other letter except for the special cases Command Description `. jump to position where last change occurred in current buffer `" jump to position where last exited current buffer `0 jump to position in last file edited (when exited Vim) `1 like `0 but the previous file (also `2 etc) '' jump back (to line in current buffer where jumped from) jump back (to position in current buffer where jumped from) [ or ] jump to beginning/end of previously changed or yanked text < or > jump to beginning/end of last visual selection
      • marking for selecting
        1. ma to mark
        2. :55 to jump to line 55
        3. shift+v to enter into visual mode
        4. 'a to go back to position a

    • suspend and un-suspend vim

      • ctrl+z to suspend
      • fg to bring it to the foreground

    • Jumping to last location

      • ```` and '' would get us to be back location
        • backtick and apostraphe can be used to jump back to the same spot on the line, or back go the beginning of the line, respectively

    • Copy and Paste

      • Shift+v - to select the entire line
      • +y - to yank it to system clipboard (tested on windows with Git Bash terminal)
      • +p - to paste it from system clipboard (tested on windows with git Bash terminal)
      • :ls - will list all the buffer

    Copy and pasting from Vim NOTE system clipboard are in the + and the * registers. To copy between vim instances, just use "ay where a is the register you want to save to

    To play back recordings

    q             # stop recording
@x            # playback to see if it works correctly
999@x         # repeat 999 times to complete the job

    typing @@ repeats the last playback

    Steps to grep, record, and playback efficiently

    :grep -r --include \*.cpp --include \*.h 'const float& name' .
# it should be in the quickfix list
Q1 # starts recording int register 1
vip>]q # visually select the paragraph, and indent. move to the next quicklist item
# it is important to move to the next quicklist item to the recording to play without
# manual interference
Q # end recording
10@1 # playback recording in register 1, 10 times
# or you can just do `@@` and it would execute and play the cursor on the next change

    The Power of G, and V

    Syntax

    :[range]g/pattern/cmd

    G and V are Opposite

    g is for ‘globally’ applying command to all lines which match the pattern v is for ‘converse’ of global, applying command to all lines that DOES NOT match the pattern

    Example

    :.,+21g/foo/d - delete any line that is between the current line, and 21 lines more, that contains foo”

    :-40,g/\v^\s+\/\//d - delete all lines from -40 to the current line that starts with \\ (c++ comments)
